## Compact Rules (inject into sub-agent prompts)

### Elixir / Mix poncho project

- Tech stack: **Elixir 1.17.3 / OTP 27**, Mix poncho project (no umbrella).
- Repo layout: 34 packages under `packages/<domain>/<name>/`, each a standalone Mix project with its own `mix.exs`, `lib/`, `test/`. Domains: `cfdi/`, `sat/`, `clir/`, `renapo/`.
- Root `mix.exs` is the **poncho aggregator**: declares each package as a `path:` dependency. Build/deps/lockfile are shared via `build_path: "../../../_build"`, `deps_path: "../../../deps"`, `lockfile: "../../../mix.lock"` in each package `mix.exs`.
- Inter-package deps use **relative paths**: same domain → `path: "../<pkg>"`, cross-domain → `path: "../../<domain>/<pkg>"`. External deps come from Hex.
- Versioning: managed via custom `mix bump` task with semver + pre-release tags + cascade. Use `--dry-run` before applying.
- Publishing: `mix hex.publish_all` publishes packages in topological order.
- Testing: ExUnit (`mix test`). Each package has its own `test/` and `test_helper.exs`. Root has `test/bump_test.exs` for the bump task.
- Formatting: `mix format` (built-in defaults — no `.formatter.exs` at root).
- No Credo, no Dialyzer, no static analysis tooling installed.

### Conventions when editing this codebase

- When adding a new package: create `packages/<domain>/<name>/{mix.exs,lib/,test/}` and register it in the appropriate function in root `mix.exs` (`cfdi_apps/0`, `sat_apps/0`, `clir_apps/0`, `renapo_apps/0`).
- Each package `mix.exs` MUST set `build_path`, `deps_path`, `lockfile` to point three levels up — otherwise the package builds in isolation instead of the shared `_build/`.
- Keep package boundaries clean: a package should only `path:`-depend on packages it directly uses. No transitive re-exporting.
- Use `async: true` in test modules unless the test mutates shared global state.
- Run tests for the single package you touched (`cd packages/<domain>/<name> && mix test`) — full sweep is `find packages -mindepth 3 -maxdepth 3 -name "mix.exs" -execdir mix test \;` per the README.

### Project-specific knowledge

- The repo is a **functional replica** of the Node.js monorepo `cfdi-node`. Package names mirror the Node `@cfdi/*`, `@sat/*`, `@clir/*`, `@renapo/*` namespaces.
- A recent migration moved everything from `apps/` to `packages/` (visible in the git status as renames). Do not introduce `apps/` paths.
